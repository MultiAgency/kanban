# Integration Philosophy and v1 Boundaries

## Scope of v1

- **GitHub‑only integration**: The initial version ships with tight integration to GitHub Issues. All core functionality (issue tracking, handoff comments, status labels) lives in the GitHub repository.
- **No generic plugin system**: A plugin architecture would add significant design and maintenance overhead without a concrete second integration request. It will be reconsidered only when a real need emerges.

## What Counts as an "Integration" in v1

1. **GitHub sync** – webhook events and periodic pull to keep the Kanban board in sync with GitHub Issues (see #20).
2. **IronClaw event ingestion** – handling of IronClaw‑generated handoff events (see #24).
3. **IronHub skill matching** – agents can be suggested based on skill tags during issue creation (see #25).

## Deferred Integrations (v1.5+)

- Linear, Notion, Asana, etc.
- Custom CRMs or generic webhooks.
- Plugin system for extensibility.
